📘 Database Design Notes

Normalization

Schema follows 3NF principles. Each table holds a single responsibility; redundant data is eliminated through foreign key references.

Indexing Strategy

Primary keys are auto-indexed. Additional indexes on tracking_number, customer_email, and status fields for fast lookups.

Audit & History

Tracking_history and payments retain immutable rows for full audit traceability across the shipment lifecycle.

Roles & Access

Users table supports admin, operator, courier, and customer roles, enforced through application-layer authorization.

Geo Routing

Branches and routes model the courier network. Shipments link origin and destination branches for route optimization.

Payments

Payments are tied to shipments via FK. Supports multiple methods: cash on delivery, card, mobile banking, wallet.
